Internet Overview

Technology components and how they fit together.

START HERE:

Your computer

This is "the client" - sends request for web page: http://www.infsoln.com

Your ISP passes your request through its gateway to the Internet. If the route to the URL is not known then must first look it up using DNS.

ISP's DNS asks: What is the address for: http://www.infsoln.com ? Domain Name Registry sends back an IP Address for the requested host to the ISP gateway. ISP gateway routes your http request to the server at the given IP Address.

infsoln.com server

The gateway at the hosting facility for infsoln.com receives the request for the web page and sends it on to the infsoln.com server.

www.infsoln.com web server

No file was specified in the request, so the web server grabs the first available default, in our example: index.php The .php file extension tells it to hand code off to the php parser

infsoln.com php

php begins following instructions in the index.php file. This file needs to get some data from the database. Using the SQL query found in the code, php sends a request to the database.

infsoln.com database server processes the query and ships back the result

www.infsoln.com web server

php completes parsing the code and turns it into html. The web server then sends the generated html code back to the infsoln.com gateway.

The infsoln.com gateway sends the html back to your ISP's gateway. Your ISP returns the result to your computer.

Your computer

Your browser receives the html code, parses it, and displays the resulting web page. Done!
