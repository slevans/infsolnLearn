Best practice is to use Cascading Style Sheets (CSS) to control both the layout 
and the look and feel of a page

They come in 3 flavors:

· remote file <- best practice;

· "in-file", where they are defined, usually in the <head> section;

· and "inline" where they are defined inside an html tag
example: <p style="color: blue;">blue text</p>

To start while you are learning we will put them in-file.

You do this with a <style> </style> tag in the head
section of the html document.

You only need one style tag, but you can have more - the style definitions in 
each will be appended to the list as if they are all within the same style tag.

Inside of style definitions are properties.  Properties are the aspects of
an html element that you wish to control, such as font size or color, etc.

Each property is listed with a semi-colon followed by the desired value. If
there is more than one property defined in a style definition they should be
separated by a semi-colon, as shown in the example below.

For example:

<head>
<style>
    #foo {
        color: red;
        font-size: 10px;
    }
</style>

<style>
    #foo {
        color: blue;
    }
</style>
</head>
<body>
<p id=foo>This text will be font size 10 and color blue.</p>
</body>


Styles are implemented using various 'CSS Selectors'.  The selectors are
applied from most general case to most specific case, regardless of what
order the style definitions are listed in the style sheet:

· 'wildcard' (*) refers to all html elements. This style does not need
to be explicity called, just defined.

· 'element' (such as h1) refers to all html tags of that type. Again,
this style does not need to be explicity called.

· 'class' (such as .boldItem) refers to a bunch of things
by the same class name.  This is defined with a period before the class
name in the style sheet, and does need to be referenced in the element tag.

· 'id' (such as #firstName) refers to one and only one
thing on a page.  This is defined in the style sheet with a hashtag
symbol before the id name, and does need to be referenced in the element tag.


When applying styles, the last property value read will be used.  In the case 
that several selectors of the same type are used, each applicable property 
will override what was previously defined in the order that they are called.


Examples:

<head>
<style>
    * {
        color: purple;
    }
    #myid {
        color: blue;
    }
    .myClass {
        color: red;
    }
    .myOtherClass {
        color: green;
    }
    h1 {
        color: orange;
    }
</style>
</head>
<body>
<h1>This will be orange</h1>
<div>This will be purple</div>
<p id="myid">This will be blue</p>
<p class="myClass">This will be red</p>
<p class="myClass" id="myid">This will be blue</p>
<p class="myClass, myOtherClass">This will be green</p>
</body>

Each html tag has different styles that will work for it, and each property
has specific values that it will accept.
