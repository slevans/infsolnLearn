HTML Basics

Html is the "language of the web". Not only is it used to contain your content and control how your content is displayed, but also determines how Search Engines perceive your page, and what scripts can run. With a little help from javascript, it accesses information contained in users' browsers and controls how those browsers will behave. Your html code is even used by third parties to determine if they want to automatically link back to you or provide a synopsis of your page.

Having a solid understanding of html and its capabilities is a required first step towards proficiency in Web development. Any code you produce for the Web will be written for the ultimate purpose of generating html.

Starting Points

  • HTML stands for Hyper-Text Markup Language.
  • This language is interpreted by web browsers.
  • HTML file names end with .html (or .htm - for Microsoft deployments).
  • HTML files are made up of tags and content.
  • All white space in the content of an html page will display as a single space, unless controlled using html tags.
  • You can view the html source code of any web page by pressing Ctrl-U

Tags

Tags represent various instructions for your browser. They can divide the page into sections and control how the content is laid out on the page. Also, tags can be given an id and other attributes or parameters that allow the content to be controlled via style sheets and javascript. The content is made up of the plain text that should be displayed on the page.

Other than the document type definition (see below) and meta tags, tags have two formats. Most tags are made up of a start tag, and an end tag (or opening/closing tag). The start tag goes before the affected content, and the end tag goes after. For example, the start tag below is "<p>" and the end tag is "</p>", which make up a "paragraph" tag:</p>

<p>This is a paragraph.</p>

Some tags however, are self-closing. Instead of a pair of start and end tags, these are made up of a single tag that is formatted like the "line break" tag:

<br />

All self-closing tags in html 5 start with a less than sign followed by the tag name, and end with "space, forward slash, greater than".

Some tags require various attributes to be set in order to function. For example one very important tag, the "anchor tag", is used to create links to other web pages or to named sections on a page. In order for the anchor tag to know where to send the user when the link is clicked it requires an "href" attribute:

<a href="http://www.infsoln.com">infsoln.com home page</a>

The text, "infsoln.com home page" will by default show in blue with an underline. When the user clicks on this underlined text, the browser will load the page found at "http://www.infsoln.com".

Most tags also accept optional attributes. These can be used to modify the behavior of the tags and the content contained therein. The value of an attribute should always be contained in quotes, as seen in the anchor tag example above.

For a complete list of tags, bookmark: http://www.w3schools.com/tags/default.asp You can click on a tag name to get the list of available attributes for that tag.

Required Tags

All HTML files (aka web pages) should have the following tags:

<!DOCTYPE html>
The document type declaration. The format to the left indicates the document type is html 5. Tells your browser which html version to use when interpreting the tags on the page. Required to be the very first line in your file.
<body> </body>
The "body tag" contains the portion of the code that DOES display on the page. A web page may only have one body tag section. Tags in this section control content display. Unlike the "head tag", the "body tag" can accept some attributes.
<head> </head>
The "head tag" contains a portion of the code that does not display on the page. A web page may only have one head tag section. Tags in this section define meta data, tell the page what supporting files to use if any, and set some page attributes.
<html> </html>
The "html tag" marks the beginning and the end of the html on the page. It can appear only once on the page, and all of your html code goes between these tags.
<meta>
The "meta tag" accepts attributes that define what meta data is being modified. While meta tags are not literally required for html to function properly, not including a unique "meta description" on each page will cause search engines to provide no description on results pages, or even a wrong one inferred by the content on the page.

Example of a good meta description:

<meta name="description" content="HTML Basics - an overview of the html language including page layout, basic concepts, required tags, and useful tags.">

<title> </title>
The "title tag" belongs in the head section. The page title appears at the top of browser windows and in browser tabs.

Sample Web Page

Below is an example of these tags put together, to comprise the minimum that is required for a properly formatted html page.

To create your first web page, paste everything between the lines below into notepad and save the file as "first.html". You may use the "File: Open" menu in your browser to find and open this file, or drag and drop the file icon into your web browser. Depending on how your desktop is configured, double clicking on the first.html file icon may also open it in your browser for you.

--------------------------------------------------------------------------------------------------

<!DOCTYPE html>

<html>

<head>

<title>HTML Basics - infsoln.com technical training</title>

<meta name="description" content="HTML Basics - an overview of the html language

including page layout, basic concepts, required tags, and useful tags.">

</head>

<body>

Hello world! <br />

<p>These documents will soon be available at the <a href="http://www.infsoln.com">infsoln.com website</a>!</p>

</body>

</html>

------------------------------------------------------------------------------------------------------------------------------

Useful Tags

<!-- -->
"Comment tag" or "Remark tag". Used to prevent html (or other) code from displaying. Remarked code will still be visible in the page's html source

Example: <p>This will display in the browser, <!-- this won't --> this will!</p>

<a> </a>
"Anchor tag". Used to create hyperlinks in your content.

<a href="http://www.infsoln.com">infsoln.com website</a>

<div> </div>
"Div tag". Used to create a section within a web page. These are commonly assigned attributes and used to apply styles and/or javascript.

<div onmouseover="javascript:doSomething();">Here is a bunch of text that will do something when you mouse over it.</div>

<img />
"Image tag". This tag is used to place images in your content.

Example: <img src="http://infsoln.com/img/infinite_logo.jpg" />

<link />
"Link tag". Used to reference resources that are not in this file. Used only in the "head tag" section.

<link rel="stylesheet" type="text/css" href="http://infsoln.com/infinite.css" />

<script></script>
"Script tag". Inserts client-side language code such as javascript into html.

There can be many of these tags on a page, and can be found in both the

head or the body sections of the page.

<script type="text/javascript">alert("this is an alert!");</script>

<span> </span>
"Span tag". Similar to a div, but by default contains no styling so is often used inline.

<p>This is normal, <span style="color: red;">this is important</span> and this isn't.</p>

<style></style>
"Style tag". Used for including style sheet (css) definitions directly in a web page. There can be many of these tags on a page, and should be placed in the head section.

<style>div { width: 300px; }</style>

<table>

<tr> <th> </th> </tr>

<tr> <td> </td> </tr>

</table>

Good for including actual tables of data, but not for formatting your page layout.

<table><tr><th>Col A</th><th>Col B</th></tr><tr><td>1</td><td>2</td></tr></table>

The above code will produce this in a browser:

Col ACol B
12
<ul>

<li> </li>

</ul>

"Unordered list" aka bullet list.

"Ordered list" is <ol> <li> </li> </ol>

Lists the "List items" (<li> </li>) preceded by bullets.

<ul><li>a</li><li>b</li><li>c</li></ul>

  • a
  • b
  • c
